# Telegram AI Bot — روی Cloudflare Workers

بات تلگرام وصل به هر API هوش‌مصنوعی سازگار با فرمت OpenAI (شامل base_url سفارشی).

## قابلیت‌ها

- **چت خصوصی:** جواب به هر پیام، با حافظه‌ی مکالمه (۱۲ turn آخر رو یادشه).
- **چت گروهی:** فقط با گفتن کلمه‌ی فراخوانی جواب میده. بعد از اولین جواب، اگه رو پیام بات **ریپلای** کنی، بدون نیاز به گفتن دوباره‌ی کلمه‌ی فراخوانی ادامه میده.
- **منوی دکمه‌ای کامل (فقط چت خصوصی):** با `/start` یا دکمه‌ی «⚙️ منو» که همیشه پایین صفحه‌ست، این کارها با لمس دکمه انجام میشه (بدون تایپ دستور):
  - **انتخاب مدل هوش‌مصنوعی** — لیست مدل‌ها **زنده از خود API** گرفته میشه (نه یه لیست ثابت که قدیمی بشه)
  - **تنظیم شخصیت بات** — رو دکمه می‌زنی، بعد یه پیام می‌فرستی که میشه شخصیت بات
  - **میزان خلاقیت پاسخ** — چهار پیش‌فرض آماده (دقیق / متعادل / خلاق / خیلی خلاق)
  - **نمایش تنظیمات فعلی**
  - **فال حافظ**
  - **پاک‌کردن حافظه‌ی گفتگو**
  - **بازگشت به پیش‌فرض**
- **`/kick`** و **`/promote`** (فقط گروه): باید رو پیام فرد موردنظر **ریپلای** کنی و دستور رو بزنی. فقط ادمین‌های واقعی گروه (طبق خود تلگرام) می‌تونن اجراش کنن؛ بات هم باید تو گروه دسترسی ادمین داشته باشه. این دو تا عمداً دستور موندن (نه دکمه) چون نیاز به مشخص‌کردن یه کاربر خاص از طریق ریپلای دارن.

## 🚀 روش خیلی ساده (بدون خط‌فرمان، فقط با کلیک)

Cloudflare یه دکمه‌ی رسمی داره که همه‌چیز رو با چندتا کلیک انجام میده — لاگین با گیت‌هاب و کلادفلر (بدون نیاز به کپی‌کردن توکن جایی)، ساخت خودکار KV، و دیپلوی. این دقیقاً همون تجربه‌ی «پنل نصب» ایه که می‌خواستی، منتها میزبانش خود Cloudflareست، نه یه سرور واسط:

[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=<[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/Aliz773/telegram-ai-bot)>)

**مراحل:**
1. اول این پروژه رو (همون‌طور که خودت گفته بودی) تو یه ریپازیتوری گیت‌هاب آپلود کن.
2. لینک همون ریپو رو تو URL دکمه‌ی بالا جایگزین کن، یا مستقیم برو به `deploy.workers.cloudflare.com` و لینک ریپوت رو اونجا پیست کن.
3. با گیت‌هاب لاگین کن، با کلادفلر لاگین کن (هر دو OAuth، هیچ توکنی جایی تایپ نمی‌کنی).
4. تو صفحه‌ی تنظیمات که Cloudflare نشونت میده، تایید کن که KV namespace ساخته بشه.
5. بزن **Deploy**.

بعد از این مرحله، فقط سه‌تا کار کوچیک با مرورگر (نه ترمینال) مونده، چون این‌ها اطلاعات شخصی‌ان که هیچ دکمه‌ای نمی‌تونه حدس بزنه:

**الف) اضافه‌کردن سه‌تا secret از پنل Cloudflare:**
برو تو داشبورد Cloudflare → پروژه‌ی تازه‌دیپلوی‌شده → **Settings** → **Variables and Secrets** → این سه‌تا رو با نوع «Secret» بساز:
- `ENCRYPTION_KEY` — یه رشته‌ی تصادفی بلند (هر چیزی، حداقل ۳۲ کاراکتر)
- `SETUP_PASSWORD` — رمزی که خودت برای صفحه‌ی تنظیمات می‌خوای
- `WEBHOOK_SECRET` — یه رشته‌ی تصادفی دیگه

بعد از اضافه‌کردن، یه‌بار از همون پنل **Deploy** رو دوباره بزن تا اعمال بشه.

**ب) پر کردن فرم تنظیمات:**
```
https://YOUR-WORKER-URL/setup
```

**ج) فعال‌سازی وبهوک:**
```
https://YOUR-WORKER-URL/api/set-webhook
```

تمام — از اول تا اینجا حتی یه‌بار هم لازم نبود ترمینال باز کنی.

## روش دوم: نصب خودکار با اسکریپت (نیاز به ترمینال و توکن API)

**⚠️ نکته‌ی امنیتی مهم:** توکن API کلادفلر رو **فقط تو ترمینال خودت** وارد کن، هیچ‌جای دیگه (چت، ایمیل، پیام) پیستش نکن — چون هرکسی این توکن رو داشته باشه می‌تونه رو اکانت Cloudflareت Worker بسازه/عوض کنه.

**ساخت توکن:** برو به `dash.cloudflare.com/profile/api-tokens` → **Create Token** → قالب **Edit Cloudflare Workers**.

### ویندوز (PowerShell)
```powershell
cd telegram-ai-bot
.\install.ps1
```
اگه خطای "اجرای اسکریپت غیرفعاله" گرفتی، یه‌بار این رو بزن:
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

### Termux / Linux / macOS
```bash
cd telegram-ai-bot
bash install.sh
```

اسکریپت در آخر آدرس Workerت رو نشون میده و می‌گه فقط دو مرحله مونده: پر کردن فرم `/setup` (چون بات‌توکن و API key شخصیه، این اسکریپت خودش نمی‌تونه حدس بزنه) و باز کردن `/api/set-webhook`.

## نصب دستی (اگه ترجیح میدی مرحله‌به‌مرحله خودت انجام بدی)

### ۱. اکسترکت و نصب
```bash
cd telegram-ai-bot
npm install
```

### ۲. لاگین
```bash
npx wrangler login
```

### ۳. ساخت KV
```bash
npx wrangler kv namespace create BOT_KV
```
مقدار `id` رو تو `wrangler.toml` جای `REPLACE_WITH_YOUR_KV_NAMESPACE_ID` بذار.

### ۴. سکرت‌ها
```bash
npx wrangler secret put ENCRYPTION_KEY
npx wrangler secret put SETUP_PASSWORD
npx wrangler secret put WEBHOOK_SECRET
```

### ۵. دیپلوی
```bash
npx wrangler deploy
```

### ۶. تنظیمات پایه
برو به `https://YOUR-WORKER-URL/setup`، فقط اطلاعات اتصال رو پر کن (توکن بات، API key، Base URL، مدل پیش‌فرض، Bot Username، کلمه‌ی فراخوانی، حداکثر طول پاسخ). شخصیت/خلاقیت/مدل دیگه از پنل تنظیم نمیشن.

### ۷. فعال‌سازی وبهوک
```
https://YOUR-WORKER-URL/api/set-webhook
```

### ۸. تنظیمات شخصی‌سازی
تو تلگرام به بات پیام خصوصی بده، `/start` بزن — یه منوی دکمه‌ای باز میشه که همه‌چیز (مدل، شخصیت، خلاقیت) از همون‌جا قابل تنظیمه.

## نکته‌ی امنیتی درباره‌ی /kick و /promote
این دستورها عمداً بر پایه‌ی ریپلای صریح + چک ادمین بودن فرستنده طراحی شدن، نه تشخیص زبان طبیعی توسط AI — چون اجازه دادن به AI برای تفسیر آزاد جمله‌هایی مثل «فلانی رو حذف کن» می‌تونه توسط اعضای غیرادمین سوءاستفاده بشه یا اشتباه تفسیر بشه.

## محدودیت‌های این نسخه
- هر Worker فقط از یک بات پشتیبانی می‌کنه.
- فقط فرمت OpenAI-compatible (`/chat/completions` و `/models`) پشتیبانی میشه. اگه provider تو endpoint لیست مدل‌ها (`/models`) نداشته باشه، یه لیست fallback کوچیک نشون داده میشه.
- حافظه‌ی مکالمه محدود به ۱۲ turn آخره (برای کنترل حجم KV).
